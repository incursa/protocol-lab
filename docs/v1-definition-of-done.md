# Public Contract Definition Of Done

ProtocolLab v1 public readiness is defined by contract clarity, not by a
repository-local runner.

## Done Means

- The public repository contains only specifications, schemas, fixtures,
  scenarios, suites, load profiles, documentation, and governance files.
- Canonical requirements, architecture, work items, and verification records
  are authored as SpecTrace JSON.
- JSON document contracts have JSON Schema coverage.
- HTTP control-plane contracts have schema or OpenAPI/YAML coverage where
  applicable.
- Public fixtures include valid, invalid, and incompatible examples where the
  distinction matters.
- Documentation makes the public/internal boundary clear.
- No page presents this repository as the canonical runner, hosted lab, package
  publisher, or executable validation tool.

## Evidence

Evidence for this definition of done is repository inspection, JSON parsing,
schema review, fixture review, and traceability review.
