# Requirements Trace

This document tracks ProtocolLab requirement IDs and status for readable traceability. Canonical ProtocolLab requirements now live under `specs/`, with the runner-owned spec artifacts grouped in their current `runner/` subfolders where applicable. This repository keeps the Markdown surface as a lightweight companion so historical planning context remains readable.

Status values: `not-started`, `partial`, `complete`, `deferred`, `unsupported`.

## Requirement ID Groups

- `PB-VISION`: project purpose, principles, and boundaries
- `PB-MODEL`: shared domain model
- `PB-MANIFEST`: implementation manifests
- `PB-SCENARIO`: scenario definitions and inventories
- `PB-VALIDATION`: validation behavior
- `PB-RUNNER`: runner orchestration and CLI flow
- `PB-LOADTOOL`: load-tool abstraction and adapters
- `PB-RESULT`: result schema and metrics
- `PB-ARTIFACT`: artifact layout and preservation
- `PB-REPORT`: reports and summaries
- `PB-SERVER`: benchmark server implementations and HTTP endpoint contract
- `PB-QUIC`: raw QUIC transport scenarios and metrics
- `PB-H3`: HTTP/3 protocol scenarios and metrics
- `PB-NETWORK`: network profiles and impairment providers
- `PB-DOCS`: documentation
- `PB-TEST`: verification and test coverage
